## Why iLink, not 公众号

公众号(包括测试号)是 **1:N 广播**:粉丝关注一个号,号给粉丝群发或回复。约束:
- 测试号 100 粉丝上限;订阅号客服消息 48h 窗口 + 模板消息白名单
- 用户「关注公众号」语义跟「派活给助理」不匹配 — 助理需要替用户给**用户的好友**发消息

iLink (`ilinkai.weixin.qq.com`) 是 Tencent 官方 AI bot 框架,通过 QR 绑用户**自己的个人微信**:
- 用户=助理共用同一个微信号
- 助理可以以「用户的口吻」给任何好友 DM
- 长轮询协议(我们主动拉,不是微信主动推),不需要公网 webhook,**绕开 ngrok / cpolar 一切隧道问题**

+ `~/workspace/mitsein/backend/apps/mitsein-api-service/bot/weixin/`

## State

### `wechat_bindings` (新表,跟 revert 掉的同名表 schema 完全不同)

```sql
CREATE TABLE wechat_bindings (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id       UUID NOT NULL UNIQUE REFERENCES users(id) ON DELETE CASCADE,
                -- UNIQUE: 一个灵犀用户只能绑一个微信号 (MVP 简化)
  ilink_bot_id  TEXT NOT NULL UNIQUE,
                -- iLink 返回的 bot id,跨 token 轮换稳定标识
  bot_token     TEXT NOT NULL,
                -- iLink 颁发,失效会触发 errcode=-14
  base_url      TEXT NOT NULL,
                -- iLink 返回的实际 API 入口 (可能 redirect)
  cursor        TEXT,
                -- get_updates_buf 长轮询游标,每次变更 UPDATE
  is_active     BOOLEAN NOT NULL DEFAULT true,
                -- session_expired 时改 false,PollManager.startAll 据此筛
  thread_id     TEXT REFERENCES threads(id) ON DELETE SET NULL,
                -- 1 用户 1 thread,首条消息时创建+回写
  bound_at      TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE INDEX idx_wechat_bindings_active ON wechat_bindings(is_active)
  WHERE is_active = true;
```

### QR 阶段的临态: **不持久化**

mitsein 让前端拿 `qrcode` (session_key) 后直接调 iLink poll,gateway 无状态中转。
我们沿用 — 简单、无 GC 负担、无重启清理问题。

## Tech notes — Python → Node 等价物

| mitsein (Python) | laifu (Node) |
|---|---|
| `asyncio.Task` | 不 await 的 async 函数 + `AbortController.signal` |
| `asyncio.create_task(...)` | `pollLoop(...).catch(...)` |
| `asyncio.sleep(n)` | 自实现 `sleep(ms, signal)` 支持 abort |
| `asyncio.CancelledError` | `AbortError` (name === 'AbortError') |
| `httpx.AsyncClient(timeout)` | `fetch(url, { signal: AbortSignal.timeout(ms) })` |
| `httpx.TimeoutException` | `fetch` 抛 `TimeoutError` (`e.name === 'TimeoutError'`) |
| 指数退避 max 30s | 一致 |
| 长轮询服务端 35s + 客户端 +5s | 一致 |

### B4. poll-loop

**Files:**
- Create: `apps/gateway/src/wechat-ilink/poll-loop.ts`
- Test: `apps/gateway/test/wechat-ilink/poll-loop.test.ts`

```ts
export const pollLoop = async (opts: {
  client: IlinkClient;
  initialCursor: string | null;
  signal: AbortSignal;
  onMessage: (raw: any) => Promise<void>;
  onCursorUpdate: (cursor: string) => Promise<void>;
  onSessionExpired: () => Promise<void>;
}): Promise<void> => {
  let cursor = opts.initialCursor;
  let consecutiveFailures = 0;
  while (!opts.signal.aborted) {
    try {
      const data = await opts.client.getUpdates(cursor, {
        timeoutMs: 35_000,
        signal: opts.signal,
      });
      consecutiveFailures = 0;
      if (data.errcode === -14) {
        await opts.onSessionExpired();
        return;
      }
      if (data.get_updates_buf && data.get_updates_buf !== cursor) {
        cursor = data.get_updates_buf;
        await opts.onCursorUpdate(cursor);
      }
      for (const msg of data.msgs ?? []) {
        try { await opts.onMessage(msg); }
        catch (e) { console.error('[poll-loop] on_message error:', e); }
      }
    } catch (e: any) {
      if (e?.name === 'AbortError') return;
      consecutiveFailures++;
      const backoffMs = Math.min(2 ** consecutiveFailures, 30) * 1000;
      console.warn('[poll-loop] poll error, backing off', backoffMs, e);
      await sleep(backoffMs, opts.signal);
    }
  }
};
```

辅助 `sleep(ms, signal)` 也要支持 abort。

**Test scenarios:**
- 正常流: getUpdates → 触 onMessage / onCursorUpdate
- errcode=-14 → onSessionExpired 调用并 return
- 异常 → 退避 → 重试
- signal.abort → loop 立即退出
- onMessage 抛错不杀循环

### B5. PollManager

**Files:**
- Create: `apps/gateway/src/wechat-ilink/poll-manager.ts`
- Test: `apps/gateway/test/wechat-ilink/poll-manager.test.ts`

```ts
export class PollManager {
  private pollers = new Map<string, AbortController>();
  constructor(private opts: {
    dao: WechatBindingDao;
    hermes: HermesClient;
    sb: SupabaseClient;
  }) {}

  async startAll(): Promise<void> {
    const bindings = await this.opts.dao.listActive();
    for (const b of bindings) this.startOne(b);
    console.log(`[PollManager] started ${bindings.length} pollers`);
  }

  startOne(binding: WechatBinding): void {
    if (this.pollers.has(binding.id)) return;
    const ac = new AbortController();
    this.pollers.set(binding.id, ac);

    const client = makeIlinkClient({
      botToken: binding.bot_token,
      baseUrl: binding.base_url,
    });

    pollLoop({
      client,
      initialCursor: binding.cursor,
      signal: ac.signal,
      onMessage: (raw) => handleInbound(binding, raw, this.opts),
      onCursorUpdate: (c) => this.opts.dao.updateCursor(binding.id, c),
      onSessionExpired: async () => {
        await this.opts.dao.deactivate(binding.id);
        this.pollers.delete(binding.id);
      },
    }).catch((e) => {
      if (e?.name !== 'AbortError') console.error('[poller crashed]', binding.id, e);
      this.pollers.delete(binding.id);
    });
  }

  stopOne(bindingId: string): void {
    const ac = this.pollers.get(bindingId);
    if (!ac) return;
    ac.abort();
    this.pollers.delete(bindingId);
  }

  async stopAll(): Promise<void> {
    for (const [, ac] of this.pollers) ac.abort();
    this.pollers.clear();
  }

  size(): number { return this.pollers.size; }
}
```

**Test scenarios:**
- `startOne` 幂等(同一 bindingId 重入不会起两个)
- `stopOne` 真的让 loop 退出
- `stopAll` 清空 map
- session_expired 自动从 map 移除
- `startAll` 启动数量 = DAO 返回的活跃数量

### B6. handleInbound: 串 hermes Agent

**Files:**
- Create: `apps/gateway/src/wechat-ilink/inbound-handler.ts`

参考 mitsein agent_bridge.py:

```ts
export const handleInbound = async (
  binding: WechatBinding,
  raw: any,
  ctx: { dao, hermes, sb }
): Promise<void> => {
  const msg = parseInbound(raw);
  if (!msg) return;                              // 非 text 跳过

  // 1. resolve thread (1 user 1 thread)
  let threadId = binding.thread_id;
  if (!threadId) {
    threadId = await ctx.sb.from('threads').insert({
      user_id: binding.user_id,
      source: 'wechat',
      title: '微信',
    }).select('id').single().then(r => r.data!.id);
    await ctx.dao.bindThread(binding.id, threadId);
  }

  // 2. 跑 hermes Agent
  const replyText = await ctx.hermes.runConversation({
    threadId,
    userId: binding.user_id,
    userMessage: msg.text,
  });

  // 3. iLink sendText
  const client = makeIlinkClient({
    botToken: binding.bot_token,
    baseUrl: binding.base_url,
  });
  await client.sendText({
    to_user_id: msg.from_user_id,
    text: replyText,
    context_token: msg.context_token,
  });
};
```

**Tests:**
- 首条消息: thread 创建 + 写回 binding.thread_id
- 后续消息: 复用 thread_id
- 非 text → silent skip,不调 hermes
- hermes 报错 → 发兜底文案给用户(「处理失败,请稍后再试」),不杀循环

1. INSERT INTO wechat_bindings (user_id, ilink_bot_id, bot_token, base_url, is_active=true)

2. 立刻 pollMgr.startOne(newBinding)
3. 返回 { bound: true, ilink_bot_id }

DELETE: 先 pollMgr.stopOne(binding.id),再 dao.deactivate(binding.id)。**不**真删行 — 保留历史(future: re-bind 同一 ilink_bot_id 时识别)。

## 已锁定的设计决策

1. **eager startAll** — gateway 启动时扫 DB,有活跃 binding 就起轮询;没就等扫码触发 startOne
2. **1 用户 1 thread** — 跟 mitsein 一致,简化 MVP,后续可分裂为 (user, contact) 对
3. **MVP 只文本** — 图片/语音/文件 unsupported,显式提示
4. **QR session 无持久化** — gateway 透传,session_key 前端自己持
5. **解绑保留行** — 不 DELETE 行,只置 is_active=false,留作历史 / 后续 re-bind 识别
